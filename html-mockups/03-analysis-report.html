研究空白

方法学局限
当前推测解码方法大多依赖于一个独立的草稿模型，这引入了额外的内存开销和调度复杂度。现有方法缺乏在主模型内部直接生成草稿 token 的轻量级机制。
支撑证据
SpecInfer 使用独立的草稿模型，需要额外 1.2B 参数的显存占用；EAGLE 虽然共享了部分参数，但仍需独立的解码头。
SpecInfer EAGLE Medusa
矛盾结论
关于 KV cache 压缩对生成品质的影响，不同论文的结论存在分歧。部分研究表明 50% 压缩率下性能无明显下降，而另一部分研究在长文本场景下观察到显著退化。
支撑证据
H2O (NeurIPS 2023) 报告在 PG-19 数据集上 60% 压缩率下 ppl 损失 < 0.1；而 SnapKV (ICLR 2024) 在 Needle-in-Haystack 测试中发现同等压缩率下长距离检索准确率下降 23%。
H2O SnapKV
未探索领域
现有推理优化研究主要关注自回归解码阶段，对 prefill 阶段的优化关注不足，特别是在长上下文场景下的注意力计算加速。
支撑证据
在已分析的 20 篇论文中，仅 3 篇涉及 prefill 阶段优化，且都聚焦于 FlashAttention 的变体，缺乏针对 KV cache 初始构建阶段的系统优化。
FlashDecoding++ vLLM
数据集不足
推理性能评估过度依赖标准基准（如 HumanEval、GSM8K），缺乏针对真实生产场景的推理负载测试集，特别是多用户并发场景下的吞吐-延迟权衡数据。
支撑证据
18/20 篇论文仅报告单请求 batch=1 的延迟指标，未覆盖在线服务中常见的动态 batch 场景。
Orca DistServe

Base 论文推荐

1
EAGLE: Speculative Sampling Requires Rethinking Feature Uncertainty
Y. Li, F. Wei, C. Zhang, H. Zhang · ICML 2024
引用: 342arXiv: 2401.15077
推荐理由
EAGLE 通过共享主干网络参数来减少草稿模型的显存开销，这一设计方向与"方法学局限"中识别的问题高度契合。该论文提供了良好的基线实现，在此基础上可以进一步探索参数共享的边界。
建议创新方向
将 EAGLE 的共享参数机制扩展为动态可配置的草稿生成层，根据输入序列的复杂度自适应调整草稿生成的深度。
2
H2O: Heavy-Hitter Oracle for Efficient Generative Inference of Large Language Models
Z. Liu, et al. · NeurIPS 2023
引用: 587arXiv: 2306.14048
推荐理由
H2O 提出了一种基于重要度评估的 KV cache 驱逐策略，其方法简洁且易于实现。在与 SnapKV 的矛盾结论中，H2O 的方案在短文本场景表现优异，适合以此为起点探索自适应压缩策略。
建议创新方向
结合 H2O 的驱逐策略与 SnapKV 的局部敏感机制，设计一种上下文长度感知的自适应 KV cache 压缩方案。
3
vLLM: Easy, Fast, and Cheap LLM Serving with PagedAttention
W. Kwon, Z. Li, et al. · 2023
引用: 891GitHub: 18k+ stars
推荐理由
vLLM 是目前最流行的 LLM 推理框架，其 PagedAttention 机制有效解决了 KV cache 的内存碎片问题。作为"数据集不足"中提到的系统性评估缺失的切入点，vLLM 的开源生态适合进行生产场景下的基准测试。
建议创新方向
在 vLLM 基础上构建面向真实生产场景的动态 batch 推理基准，研究不同并发模式下的吞吐-延迟最优配置。

低难度 方法改进
在 EAGLE 的基础上引入动态草稿深度控制：根据当前 token 的预测不确定性（entropy），自适应决定生成草稿 token 的数量（1-5 个），而非固定的草稿长度。

## 新增需求

### 需求: 项目仪表盘
前端必须显示所有研究项目的列表，包含名称、主题、状态和创建日期。用户可以创建新项目或导航到已有项目。

#### 场景: 查看项目列表
- **当**用户打开应用时
- **则**系统按创建日期倒序（最新的在前）展示项目列表

#### 场景: 空项目列表
- **当**用户没有任何项目时
- **则**系统显示空状态组件，提示"还没有项目，创建一个开始研究"，并展示"新建项目"按钮

#### 场景: 从仪表盘创建新项目
- **当**用户点击"新建项目"并填写名称和主题时
- **则**项目创建成功，用户被导航到项目详情页

#### 场景: 创建项目表单校验
- **当**用户提交创建项目时名称或主题为空
- **则**系统显示校验错误提示，不发送请求

#### 场景: 从仪表盘删除项目
- **当**用户点击项目的删除按钮并确认时
- **则**项目被逻辑删除，列表刷新

### 需求: 前端路由定义
前端必须定义以下路由：`/` 项目仪表盘，`/projects/{id}` 项目详情（论文检索 + 分析触发 + Base 论文选择），`/projects/{id}/analysis` 分析报告。

#### 场景: 访问项目详情页
- **当**用户访问 `/projects/{id}` 时
- **则**系统展示项目信息、已检索论文列表、"搜索论文"按钮和"触发分析"按钮

#### 场景: 访问分析报告页
- **当**用户访问 `/projects/{id}/analysis` 时
- **则**系统展示 LLM 生成的分析报告，包含研究空白、base 论文推荐和创新点

#### 场景: 分析报告未触发过
- **当**用户访问分析报告页但该项目从未触发过分析时
- **则**系统显示空状态，提示"尚未进行分析"，并提供"返回项目详情"按钮

### 需求: 面包屑导航
前端必须使用面包屑组件在 Dashboard → ProjectDetail → AnalysisReport 之间提供导航。

#### 场景: 项目详情页面包屑
- **当**用户访问 `/projects/{id}` 时
- **则**面包屑显示"项目列表 > {项目名}"，点击"项目列表"返回 Dashboard

#### 场景: 分析报告页面包屑
- **当**用户访问 `/projects/{id}/analysis` 时
- **则**面包屑显示"项目列表 > {项目名} > 分析报告"，可点击返回上级页面

### 需求: 论文搜索界面
前端必须提供搜索表单，用户可输入关键词、触发检索，并在分页列表中查看结果。

#### 场景: 执行论文搜索
- **当**用户输入关键词并点击"搜索"时
- **则**系统显示加载指示器，然后展示论文结果列表，包含标题、摘要、作者、引用数和代码可用性

#### 场景: 搜索时状态控制
- **当**项目处于 SEARCHING 或 ANALYZING 状态时
- **则**搜索按钮禁用，不可重复触发检索

#### 场景: 查看论文详情
- **当**用户在结果列表中点击一篇论文时
- **则**系统从右侧滑出抽屉展示论文详情，包括完整摘要、发表场所和 PDF/代码链接

#### 场景: 部分失败提示
- **当**检索成功但某一数据源失败时
- **则**系统在论文列表顶部展示黄色警告条，说明哪个数据源不可用

### 需求: 论文列表多选
前端必须支持用户勾选部分论文进行后续操作（分析、查看）。

#### 场景: 勾选论文
- **当**用户在论文列表中勾选复选框时
- **则**系统更新已选数量显示（如"已选: 3/42 篇"）

#### 场景: 全选/取消全选
- **当**用户点击全选复选框时
- **则**当前页所有论文被勾选或取消勾选

### 需求: 分析触发与进度展示
前端必须在 ProjectDetail 页面内嵌展示分析任务的实时进度。

#### 场景: 触发异步分析
- **当**用户点击"触发分析"按钮时
- **则**按钮禁用，页面底部显示内嵌进度面板

#### 场景: 分析进度展示
- **当**分析任务执行中时
- **则**内嵌进度面板显示进度条、阶段指示器（如"阶段 1/3: 正在逐篇深度理解... (9/20)"）

#### 场景: 分析完成后跳转
- **当**分析任务完成后
- **则**进度面板显示完成状态，用户可点击"查看报告"跳转

#### 场景: 分析状态控制
- **当**项目处于 CREATED 状态（未检索）时
- **则**"触发分析"按钮不可见
- **当**项目处于 ANALYZING 状态时
- **则**"触发分析"按钮禁用

### 需求: Base 论文选择
前端必须在 ProjectDetail 页面提供 Base 论文选择入口，从已检索论文中选择一篇。

#### 场景: 打开 Base 论文选择
- **当**用户点击"选择 Base 论文 →"时
- **则**系统展示 Base 论文选择面板，列出所有已检索论文

#### 场景: 确认 Base 论文
- **当**用户在面板中选择一篇论文并点击"确认选择"时
- **则**该论文被关联到项目，ProjectDetail 顶部显示已选 Base 论文摘要卡片

#### 场景: 已选 Base 论文展示
- **当**项目已有 base_paper_id 时
- **则**ProjectDetail 顶部显示 Base 论文卡片（标题、作者、引用数），并提供"更换"按钮

#### 场景: 更换 Base 论文
- **当**用户点击"更换 Base 论文"时
- **则**系统重新打开 Base 论文选择面板

### 需求: 分析结果展示
前端必须以结构化、可读的格式展示 LLM 生成的分析结果，包括研究空白、推荐 base 论文和创新点。

#### 场景: 查看分析报告
- **当**项目的分析完成后
- **则**系统分区域展示研究空白、base 论文推荐和创新点

#### 场景: Base 论文未选中时的创新点
- **当**用户未选择 Base 论文但已有分析结果时
- **则**系统在创新点区域顶部提示"选择 Base 论文后可获得更精准的创新建议"

### 需求: WebSocket 任务进度监听
前端必须通过 STOMP 协议订阅 `/topic/task/{taskId}` 接收异步任务的实时进度更新。

#### 场景: 监听检索任务进度
- **当**用户触发论文检索后
- **则**前端订阅 `/topic/task/{taskId}`，根据推送消息更新 UI 中的进度条和阶段提示

#### 场景: 监听分析任务进度
- **当**用户触发创新分析后
- **则**前端订阅 `/topic/task/{taskId}`，根据推送消息更新 UI 中的进度条和阶段提示

#### 场景: WebSocket 断线重连
- **当**WebSocket 连接断开时
- **则**前端自动重连，重连期间轮询任务状态接口作为降级方案

#### 场景: WebSocket 断线期间轮询降级
- **当**WebSocket 连接不可用时
- **则**前端使用 HTTP 轮询 `GET /api/v1/papers/tasks/{taskId}` 或 `GET /api/v1/analysis/tasks/{taskId}` 获取任务状态

#### 场景: STOMP 心跳
- **当**WebSocket 连接建立后
- **则**前端与服务端交换 STOMP 心跳（发送 30s 间隔，期望接收 30s 间隔），心跳超时时触发断线重连

### 需求: 统一 API 错误处理
前端必须拦截所有 HTTP 响应，解析后端 `ApiResponse` 格式中的 code 字段，统一展示错误消息。

#### 场景: 后端返回业务错误
- **当**后端返回 `{ code: 4001, message: "关键词不能为空" }` 时
- **则**前端弹出 Element Plus Message 组件展示错误消息，不显示系统崩溃页面

#### 场景: 网络异常
- **当**前端无法连接到后端服务器时
- **则**前端展示网络连接错误提示，并提供重试按钮

### 需求: UI 设计系统约束
前端页面开发必须严格复用 `html-mockups/` 目录下的高保真 HTML mockup 设计，不得自行修改颜色、间距、字体、布局。

#### 场景: Dashboard 页面 UI
- **当**用户打开项目仪表盘时
- **则**页面布局、项目卡片样式、状态标签、按钮样式、新建项目对话框、删除确认对话框均与 `html-mockups/01-dashboard.html` 一致

#### 场景: ProjectDetail 页面 UI
- **当**用户访问项目详情页时
- **则**页面布局、Base 论文卡片、搜索区、论文表格、排序切换、分页、操作按钮、任务进度面板、论文详情抽屉、Base 论文选择器均与 `html-mockups/02-project-detail.html` 一致

#### 场景: AnalysisReport 页面 UI
- **当**用户访问分析报告页时
- **则**页面布局、研究空白卡片网格、Base 推荐列表、创新点网格、面包屑导航均与 `html-mockups/03-analysis-report.html` 一致

#### 场景: 设计系统变量复用
- **当**前端页面开发时
- **则**颜色变量 (--accent: #2563eb, --danger: #dc2626 等)、圆角变量 (--radius-sm: 8px, --radius-md: 10px, --radius-lg: 12px)、阴影变量、字体族、字号、间距必须与 mockup 中的 CSS Custom Properties 一致

### 需求: 加载与错误状态
前端必须在异步操作（搜索、分析）期间显示适当的加载指示器，并为失败的请求显示错误消息和重试选项。

#### 场景: 搜索期间显示加载状态
- **当**论文搜索正在进行时
- **则**系统显示进度转圈图标并禁用搜索按钮

#### 场景: 分析期间显示加载状态
- **当**创新分析正在进行时
- **则**系统在内嵌进度面板中展示进度并禁用分析按钮

#### 场景: 处理 API 错误
- **当**后端返回错误响应时
- **则**系统显示错误消息并提供重试按钮

## 新增需求

### 需求: 创建研究项目
系统必须允许用户创建新的研究项目，包含项目名称、描述和研究主题/关键词。

#### 场景: 使用必填字段创建项目
- **当**用户提供项目名称和研究主题时
- **则**系统创建新项目并返回唯一的项目 ID

#### 场景: 缺少名称时创建项目
- **当**用户尝试创建没有名称的项目时
- **则**系统返回验证错误

### 需求: 更新研究项目
系统必须允许用户更新项目的名称、描述和研究主题。

#### 场景: 更新项目基本信息
- **当**用户提供有效的项目 ID 和更新的字段时
- **则**系统更新项目信息并返回更新后的项目详情

### 需求: 设置 Base 论文
系统必须允许用户为项目选择一篇 base 论文作为后续创新的起点。

#### 场景: 设置 base 论文
- **当**用户提供有效的项目 ID 和论文 ID 时
- **则**系统将论文的 ID 保存到项目的 base_paper_id 字段

#### 场景: 设置不存在的论文为 base 论文
- **当**用户指定的论文 ID 不存在时
- **则**系统返回 404 错误

### 需求: 查看项目详情
系统必须展示项目的元数据、关联论文、分析结果和当前状态。

#### 场景: 查看已有项目
- **当**用户导航到一个项目时
- **则**系统返回项目名称、主题、创建日期、状态、论文列表和分析结果

#### 场景: 查看不存在的项目
- **当**用户请求一个不存在的项目 ID 时
- **则**系统返回 404 错误

### 需求: 项目列表分页
系统必须对项目列表进行分页，支持可配置的页大小（默认 20），并按创建日期倒序返回。

#### 场景: 获取第一页项目列表
- **当**用户请求项目列表且未指定页码时
- **则**系统返回前 20 个项目，按创建日期倒序，附带总记录数

#### 场景: 获取后续页项目列表
- **当**用户请求第 2 页且页大小为 10 时
- **则**系统返回第 11-20 个项目

### 需求: 删除研究项目
系统必须允许用户逻辑删除项目，标记其所有关联的论文和分析结果为已删除状态，数据保留但不再出现在查询结果中。

#### 场景: 逻辑删除已有项目
- **当**用户删除一个项目时
- **则**系统将该项目及其关联的 paper、analysis_result 标记为 `is_deleted = 1`，数据保留在数据库中

#### 场景: 删除不存在的项目
- **当**用户尝试删除一个不存在的项目时
- **则**系统返回 404 错误

### 需求: 项目状态追踪
系统必须追踪并更新项目的工作流状态，包括：CREATED（已创建）、SEARCHING（检索中）、SEARCHED（检索完成）、ANALYZING（分析中）、ANALYZED（分析完成）。

#### 场景: 追踪状态转换
- **当**用户在 CREATED 状态的项目上触发论文检索时
- **则**项目状态更新为 SEARCHING，完成后更新为 SEARCHED

#### 场景: 追踪分析状态
- **当**用户在 SEARCHED 状态的项目上触发分析时
- **则**项目状态更新为 ANALYZING，完成后更新为 ANALYZED

#### 场景: 非法状态转换
- **当**用户尝试在 CREATED 状态的项目上触发分析时
- **则**系统返回错误，提示必须先完成论文检索

#### 场景: 状态枚举约束
- **当**更新项目状态时
- **则**系统仅接受预定义的状态值（CREATED / SEARCHING / SEARCHED / ANALYZING / ANALYZED），拒绝非法值

### 需求: 逻辑删除级联策略
系统必须在删除项目时级联标记其关联数据为已删除，数据保留在数据库中但不再出现在查询结果中。

#### 场景: 删除项目时级联标记
- **当**用户删除一个项目时
- **则**系统将该项目及其关联的 paper 和 analysis_result 记录的 is_deleted 字段标记为 1

#### 场景: 逻辑删除后查询不可见
- **当**查询被逻辑删除的项目时
- **则**系统不再返回该项目及其关联数据，即使数据库中记录仍存在

#### 场景: 删除不存在的项目
- **当**用户尝试删除一个不存在的项目时
- **则**系统返回 404 错误

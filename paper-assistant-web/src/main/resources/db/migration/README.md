# SQL 迁移命名规范

## 命名规则

- 格式: `V{版本号}__{描述}.sql` (双下划线)
- 示例: `V1__init.sql`, `V2__add_llm_config.sql`
- 版本号单调递增，从 V1 开始

## 已执行脚本不可修改

- 一旦 Flyway 标记某个迁移为已执行，该脚本**禁止修改**
- 修改已执行脚本会导致 `validate-on-migrate` 校验失败，应用启动报错

## 新增迁移

- 只能**新增**迁移文件，不能修改已有文件
- 版本号必须大于当前最大版本号
- 新迁移会自动在下一个应用启动时执行

## 回滚策略

- Flyway 不支持自动回滚
- 手动回滚：编写 `V{版本号}__UNDO__描述.sql` 迁移文件
- 或通过数据库管理工具手动回退

## 开发环境注意事项

- 开发阶段如果发现迁移脚本有 bug：
  - 如果脚本**尚未提交到版本控制**且**尚未在任何环境执行**，可以直接修改
  - 如果已执行，必须新增迁移来修复
